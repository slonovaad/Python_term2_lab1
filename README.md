## Алгоритм решения
  1. Пользователь вводит в терминал команду, программа записывает введённую команды в лог.
  2. Если введённая команда - ```exit```, программа прекращает работу.
  3. Если введённая команда - ```make_source```, программа запрашивает тип источника и вызывает метод ```make_source_by_stdin``` класса, соответсвующего указанному типу.
  4. Если введённая команда - ```get_task``` или ```get_all_tasks```, программа запрашивает имя источника и вызывает метод ```get_task``` или ```get_all_tasks``` соответственно у указанного источника.
  5. Если введена неизвестная команда, выводится и записывается в лог сообшение об этом.
  6. Если в ходе выполнения функции произошла ошибка источника или класса, выводится и записывается в лог сообшение об этом.
  7. Возвращение к шагу 1.
  8. При нажатии пользователем Ctrl+c программа завершает работу

## Инструкция к использованию

### Запустить
```python -m src.main```

### Остановить программу
```exit```

### Формат ввода
Каждая команда/аргумент представляют из себя одну строку, вводимую пользователем.
После ввода команды при нажатии enter по очереди по одному запрашиваются аргументы.

### Команды и синтаксис
1. ```make_source```
    - Создаёт новый источник
    - Запрашивает тип источника: ```file```, ```generator``` или ```stdin```
    - Далее в зависимости от типа запрашиваются необходимые аргументы
    - Ничего не выводит
2. ```get_task```
    - Получает одну задачу из источника
    - Запрашивает имя источника
    - Далее в зависимости от типа указанного источника запрашиваются необходимые аргументы
    - Выводит информацию о задаче
3. ```get_all_tasks```
    - Получает все задачи из источника
    - Запрашивает имя источника
    - Далее в зависимости от типа указанного источника запрашиваются необходимые аргументы
    - Выводит список задач
4. ```exit```
    - Осуществляет выход из программы
